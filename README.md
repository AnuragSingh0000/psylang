# Psylang

Psylang is a custom statically and dynamically typed programming language. Currently, the lexical analyzer, parser, Abstract Syntax Tree (AST) generation, **Semantic Analyzer**, and **Intermediate Representation (IR) Quadruple Generator** have been implemented.

IR operation contracts are documented in `IR_OPERATIONS.md`.

## Project Structure

- `src` - Lexer (`psy.l`) and parser (`psy.y`) specifications, AST definitions (`defs.h`), **Semantic Analyzer (`semantic.c`)**, **IR Generator (`ir_gen.c`)**, **IR Dumper (`ir_dump.c`)**, code to print the derivation tree, along with the generated C code.
- tests` - Sample `.psy` files for testing language features, organized into success, lexer failures, parser failures, **semantic failures**, and the 10 required example programs.
- `results` - Directory containing the outputs and expected outputs from running the test suites.
- `LangSpec.pdf` - Language Specification Document.

## Build and Usage

To compile the project:
```bash
make
```

To run a specific file, perform semantic analysis, and dump the **Intermediate Representation (IR) table** along with the Type fields table and Symbol Table:
```bash
DUMP_IR=1 ./src/psy.out path/to/file.psy
```

To run a file and dump the IR along with the **Symbol Table**:
```bash
DUMP_IR=1 PSY_DEBUG=1 ./src/psy.out path/to/file.psy
```

To run a specific file and print the AST:
```bash
make run-parser FILE=path/to/file.psy
```

To print the reverse derivation tree:
```bash
make run-print-rdt FILE=path/to/file.psy
```

To print out the stream of tokens generated by the lexer:
```bash
make run-print-tokens < path/to/file.psy
```

To run all the test suites (Lexer, Parser, Semantic Analyzer, and IR outputs):
```bash
make test-all
```
This automatically traverses the `tests` directory, updates the `results` folder, compares outputs against expected results, and reports the number of passing tests.

To print the parsing table:
```make
cd src
python generate_table.py
```
This outputs `LALR_TABLE.csv` in the root directory of this project.

## Error Handling

The compiler has comprehensive diagnostic reporting. It catches lexical errors (e.g., unknown tokens, unterminated strings), syntactic errors (e.g., unexpected tokens, missing semicolons), and **semantic errors** (e.g., type mismatches, undeclared variables, missing return paths).

Errors pinpoint the exact line and column of the issue and provide an indicator pointing to the problematic code.

### Parser Error Example
Input:
```text
var int a = 5;
var int b = 3

echo a + b;
```

Output:
```text
error: syntax error, unexpected echo, expecting ;
  --> tests/failure/parser/missing_semicolon.psy:4:5
   |
  4| echo a + b;
   |    ^--- Here
```

### Lexer Error Example
Input:
```text
var 9abcd = 10;
echo 9abcd;
```

Output:
```text
[Error at Line 1]: Identifier cannot begin with a digit: 9abcd
```

### Semantic Error Example
Input:
```text
var bool is_valid = true;
is_valid = 1;
```

Output:
```text
Parsed OK
error: Type mismatch in assignment.
  --> tests/failure/resolver/mismatched_type_assignment.psy:2:1
   |
  2| is_valid = 1;
   | ^--- Here


Semantic analysis failed. Compilation stopped.
```
